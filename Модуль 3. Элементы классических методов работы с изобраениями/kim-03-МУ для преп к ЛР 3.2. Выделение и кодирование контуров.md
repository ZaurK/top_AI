### 4. Рекомендации по проведению отдельных разделов

#### 4.1. Предварительная обработка изображения

**Ключевые моменты для объяснения:**

1. **Инверсия яркости** — важный шаг, поскольку алгоритм прослеживания обычно работает с белыми объектами на чёрном фоне. Научите студентов автоматически определять необходимость инверсии:

```python
# Автоматическое определение необходимости инверсии
if np.mean(img) < 127:  # если изображение в основном тёмное
    img = 255 - img
```

2. **Выбор фильтра** — обсуждается разница между гауссовским и медианным фильтром:
   - Гауссовский — лучше для сглаживания, но может размывать границы.
   - Медианный — лучше сохраняет границы, эффективно подавляет импульсный шум.

3. **Выбор порога бинаризации** — важно подчеркнуть, что качество выделения контуров критически зависит от порога. Метод Оцу (`cv2.THRESH_OTSU`) подходит для большинства случаев, но не является универсальным.

**Типичные ошибки:**
- Использование слишком сильного размытия, приводящее к потере мелких деталей.
- Неправильный выбор порога (слишком высокий — объекты разрываются, слишком низкий — появляются ложные контуры).

**Рекомендация:** предложить студентам визуализировать результаты при разных параметрах и выбрать оптимальные.

#### 4.2. Выделение контуров

**Сравнение подходов:**

| Подход | Преимущества | Недостатки |
|--------|--------------|------------|
| `cv2.findContours` | Быстро, оптимизировано, надёжно | "Чёрный ящик", трудно модифицировать |
| Собственная реализация алгоритма Розенфельда | Полный контроль над процессом, образовательная ценность | Медленнее, требует внимания к деталям |

**Рекомендация:** предложить сильным студентам реализовать алгоритм Розенфельда самостоятельно, а остальным — использовать `findContours`, но с обязательным преобразованием в цепной код.

**Важное замечание:** в `cv2.findContours` нужно использовать `cv2.CHAIN_APPROX_NONE`, чтобы получить все точки контура, а не упрощённую версию.

#### 4.3. Цепной код Фримена

**Методический приём:** предложите студентам вручную пройти по небольшому контуру (например, 3×3 пикселя) и записать цепной код. Это помогает понять логику алгоритма.

**На что обратить внимание:**
- Различие между 4-связным и 8-связным кодом. Для задач распознавания чаще используется 8-связный.
- Проблема выбора начальной точки и направления обхода — код неинвариантен к этим параметрам. Объясните, что для инвариантности нужно нормализовать код (например, начинать с минимального направления).

```python
def normalize_chain_code(chain):
    """
    Нормализация цепного кода (начало с минимального направления).
    """
    # Поиск минимального элемента
    min_idx = min(range(len(chain)), key=lambda i: chain[i])
    # Циклический сдвиг
    return chain[min_idx:] + chain[:min_idx]
```

#### 4.4. Эквализация кода контура

**Самый сложный этап работы.** Рекомендуется:

1. Дать чёткое математическое описание алгоритма.
2. Показать блок-схему.
3. Предложить использовать интерполяцию для вычисления промежуточных точек.

**Альтернативный подход:** использование `cv2.approxPolyDP` для аппроксимации контура многоугольником с фиксированным числом вершин:

```python
epsilon = 0.01 * cv2.arcLength(contour, True)
approx = cv2.approxPolyDP(contour, epsilon, True)
# Затем преобразовать вершины многоугольника в цепной код
```

Этот подход проще, но менее точен при малом числе вершин.

**Важно:** эквализация — это не просто уменьшение числа точек, а равномерная дискретизация по длине дуги. Это ключевое отличие, которое нужно подчеркнуть.

#### 4.5. Комплекснозначное кодирование и геометрические преобразования

**Демонстрационный приём:** покажите на доске, как умножение на комплексное число $e^{j\theta}$ поворачивает вектор. Затем свяжите это с поворотом всего контура.

**Важное предостережение:** комплекснозначный код не инвариантен к изменению начальной точки. Для достижения инвариантности требуется дополнительная нормализация (например, центрирование и поворот к главной оси).

### 5. Типичные ошибки и способы их предотвращения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Контуры не выделяются или выделяются с ошибками | Неправильная бинаризация (порог) | Проверить значения порога, использовать визуализацию |
| Цепной код содержит ошибки (-1) | Вектор перемещения не соответствует 8-связному соседству | Проверить, что контур замкнутый, использовать `cv2.CHAIN_APPROX_NONE` |
| После эквализации форма сильно искажается | Слишком малое значение S | Увеличить S, оценить качество визуально |
| Комплексный код даёт неверные координаты после поворота | Ошибка в формуле поворота | Проверить, что поворот применяется к каждому вектору, а не к координатам точек |
| Программа зависает при выделении контуров | Бесконечный цикл в алгоритме Розенфельда | Добавить защиту от зацикливания (максимальное число итераций) |
| Неправильное сохранение банка контуров | Ошибка сериализации комплексных чисел | Сохранять действительную и мнимую части отдельно |

### 8. Контрольные вопросы для защиты (с ответами для преподавателя)

1. **Почему для выделения контуров используется бинаризация?**  
   *Ответ:* Алгоритмы прослеживания работают с бинарными изображениями, где объект и фон чётко разделены. Это упрощает определение принадлежности пикселя.

2. **В чём отличие 4-связного и 8-связного цепного кода?**  
   *Ответ:* 4-связный код использует 4 направления (вверх, вниз, влево, вправо), 8-связный — 8 (добавлены диагональные). 8-связный точнее описывает контур, но чувствительнее к шуму.

3. **Почему эквализация необходима для распознавания?**  
   *Ответ:* Объекты могут иметь разные размеры, и их контуры будут иметь разную длину. Эквализация приводит все контуры к единой размерности, что позволяет сравнивать их между собой.

4. **Как выполняется поворот контура в комплексном представлении?**  
   *Ответ:* Каждый комплексный вектор кода умножается на $e^{j\theta}$, где $\theta$ — угол поворота. Это соответствует повороту каждого элементарного вектора.

5. **Инвариантен ли комплексный код к изменению начальной точки обхода?**  
   *Ответ:* Нет. Для достижения инвариантности требуется дополнительная нормализация (например, поиск начала с минимальной фазой).

6. **Как оценить качество эквализации?**  
   *Ответ:* Визуально — сравнить исходный и эквализованный контуры. Количественно — вычислить среднеквадратичную ошибку между точками.

7. **Что такое банк эталонных контуров и для чего он нужен?**  
   *Ответ:* Банк контуров — это набор эталонных контуров для каждого класса объектов. Он используется для сравнения с контуром неизвестного объекта в задаче распознавания.

8. **Какие параметры влияют на результат эквализации?**  
   *Ответ:* Размерность S, метод интерполяции (линейная, кубическая), исходная длина контура L.

9. **Почему в алгоритме Розенфельда используется обход по часовой стрелке?**  
   *Ответ:* Это обеспечивает детерминированность алгоритма — при одинаковых условиях он всегда даёт один и тот же результат.

10. **Как можно ускорить алгоритм выделения контуров для больших изображений?**  
    *Ответ:* Использовать `cv2.findContours` (оптимизированный на C++), уменьшить разрешение изображения, использовать распараллеливание.
