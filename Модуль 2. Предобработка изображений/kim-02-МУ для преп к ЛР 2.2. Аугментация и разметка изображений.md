---

### 1. Общие положения

**Дисциплина:** Компьютерное зрение / Глубокое обучение / Подготовка данных для машинного обучения.

**Трудоёмкость:** 4–6 академических часов (два занятия).

**Уровень сложности:** Средний (требует базовых знаний Python, работы с изображениями и понимания задач компьютерного зрения).

**Место в курсе:** Работа знакомит студентов с критически важным этапом подготовки данных для моделей компьютерного зрения. Она может выполняться как перед изучением архитектур нейросетей, так и параллельно с ними.

**Пререквизиты:**
- Базовое знание Python.
- Понимание задач компьютерного зрения (классификация, обнаружение объектов, сегментация).
- Навыки работы с библиотеками OpenCV и NumPy.

### 2. Дидактические цели и задачи

**Образовательные:**

- Научить применять аугментацию для увеличения обучающей выборки.

**Развивающие:**
- Развить навыки работы с веб-инструментами и API.
- Сформировать умение выбирать подходящий инструмент в зависимости от задачи.
- Научить писать скрипты для автоматизации повторяющихся операций.

**Воспитательные:**
- Привить культуру документирования данных и воспроизводимости экспериментов.
- Подчеркнуть важность качества данных для успеха проектов в области ИИ.

### 3. Организационные рекомендации

#### 3.1. Материально-техническое обеспечение

- Компьютерный класс с выходом в интернет.
- Установленный Python 3.8+ с библиотеками: `opencv-python`, `numpy`, `matplotlib`, `albumentations`.
- Браузер (Chrome, Firefox) для работы с VIA и CVAT.

- Рекомендуется предоставить студентам единый набор изображений для разметки (например, изображения с кошками и собаками, или с различными типами транспортных средств).

### 4. Рекомендации по проведению отдельных разделов

#### 4.1. Разметка в VIA

- **Ключевые моменты:** VIA не требует установки – достаточно открыть HTML-файл. Это важно подчеркнуть, так как студенты могут сразу приступить к работе.
- **Рекомендация:** Предложить студентам разметить не менее 5–10 изображений, используя как минимум два типа аннотаций (прямоугольники и полигоны).
- **Типичные ошибки:** Забывают сохранить аннотации перед закрытием вкладки; не добавляют атрибуты классов.

#### 4.2. Разметка в CVAT

- **Ключевые моменты:** CVAT требует регистрации на [app.cvat.ai](https://app.cvat.ai) или локальной установки. Для упрощения рекомендуется использовать облачную версию.
- **Рекомендация:** Показать, как создавать проекты, задачи, настраивать лейблы и экспортировать данные в разных форматах.
- **Типичные ошибки:** Неправильная настройка лейблов; путаница между проектом и задачей; проблемы с экспортом из-за несоответствия формата.

#### 4.3. Аугментация с Albumentations

- **Ключевые моменты:** Важно показать, как аугментировать не только изображения, но и аннотации.
- **Рекомендация:** Дать готовый пример пайплайна с разными типами аугментаций и объяснить параметры каждого преобразования.
- **Типичные ошибки:** Забывают передать `bbox_params` в `Compose`; не учитывают изменение координат после аугментации.

#### 4.4. Сравнительный анализ

- **Рекомендация:** Предложить студентам заполнить таблицу сравнения по следующим критериям: установка, интерфейс, функциональность, скорость, форматы экспорта, возможность командной работы.
- **Обсуждение:** В ходе защиты обсудить, для каких сценариев какой инструмент подходит лучше.

#### 4.5. Автоматизация

- **Рекомендация:** Скрипт автоматизации должен быть простым, но функциональным: загрузка аннотаций, применение аугментаций, сохранение результата. Это развивает навыки программирования и понимание форматов данных.

### 5. Типичные ошибки и способы их предотвращения

| Ошибка | Решение |
|--------|---------|
| Потеря аннотаций в VIA | Напомнить о необходимости экспорта перед закрытием вкладки. |
| Неправильный формат аннотаций при экспорте из CVAT | Показать, как выбирать нужный формат (COCO, YOLO, VOC). |
| Аугментация без учёта аннотаций | Объяснить, что для каждой аугментации нужно передавать параметры аннотаций. |
| Медленная работа CVAT при большом количестве изображений | Рекомендовать разбивать на несколько задач. |
| Проблемы с установкой библиотек | Предоставить файл `requirements.txt`. |

### 6. Индивидуальные задания

- **Вариант 1:** Использовать VIA для разметки 10 изображений, CVAT для 10 изображений, сравнить время и качество.

### 8. Контрольные вопросы для защиты (с ответами)

1. **Чем отличается VIA от CVAT?**  
   *Ответ:* VIA – лёгкий, не требующий установки инструмент для индивидуальной работы. CVAT – профессиональная платформа с поддержкой командной работы, видео, автоматизации и экспортом в различные форматы.

2. **Какие форматы аннотаций поддерживаются?**  
   *Ответ:* VIA экспортирует в CSV и JSON. CVAT поддерживает COCO, YOLO, Pascal VOC, и многие другие.

3. **Зачем нужна аугментация данных?**  
   *Ответ:* Чтобы увеличить объём обучающей выборки, улучшить обобщающую способность модели и сделать её устойчивой к различным искажениям (повороты, освещение, шум).

4. **Как Albumentations обеспечивает согласованность аугментаций?**  
   *Ответ:* Используя параметр `bbox_params` или `keypoint_params` в `Compose`, библиотека автоматически преобразует аннотации в соответствии с применёнными геометрическими преобразованиями.

5. **Что такое пайплайн аугментаций?**  
   *Ответ:* Это последовательность преобразований, применяемых к изображению (и аннотациям) с заданными вероятностями. Позволяет создавать разнообразные вариации данных.

6. **Какие задачи решаются с помощью разметки и аугментации?**  
   *Ответ:* Подготовка данных для обучения моделей обнаружения объектов, сегментации, классификации, распознавания ключевых точек и других задач компьютерного зрения.

### 10. Заключительные рекомендации

1. **Поощряйте самостоятельное исследование.** Пусть студенты экспериментируют с разными аугментациями и форматами экспорта.
2. **Обращайте внимание на качество разметки.** Объясните, что даже лучшая модель не сможет компенсировать плохие данные.
3. **Связывайте с реальными проектами.** Обсудите, как подготовка данных организована в индустрии, какие инструменты используются в крупных компаниях.
4. **Используйте обратную связь.** После выполнения работы обсудите со студентами, какие инструменты им показались удобнее и почему.
