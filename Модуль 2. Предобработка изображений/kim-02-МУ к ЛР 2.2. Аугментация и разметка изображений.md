### 2. Теоретические сведения

#### 2.1. Роль разметки и аугментации в компьютерном зрении

Качество моделей компьютерного зрения напрямую зависит от качества и объёма обучающих данных. **Разметка** (аннотирование) изображений – это процесс присвоения меток объектам на изображениях (ограничивающие рамки, полигоны, ключевые точки, семантические сегменты). **Аугментация** – это процесс искусственного увеличения обучающей выборки путём применения различных преобразований к исходным изображениям и их аннотациям (повороты, масштабирование, изменение яркости, добавление шума и т.д.).

#### 2.2. Инструменты разметки

**VGG Image Annotator (VIA)** – это лёгкий, не требующий установки инструмент разметки, работающий в веб-браузере. VIA разработан Visual Geometry Group (VGG) Оксфордского университета и распространяется под лицензией BSD-2. VIA позволяет:
- Загружать изображения и видео;
- Создавать аннотации в виде прямоугольников, полигонов, окружностей, эллипсов, точек и линий;
- Добавлять текстовые атрибуты к областям;
- Экспортировать аннотации в форматы CSV и JSON.

**CVAT (Computer Vision Annotation Tool)** – это профессиональный веб-ориентированный инструмент для аннотирования изображений и видео, разработанный для работы с большими проектами и командами. CVAT поддерживает:
- Аннотирование изображений и видео (обнаружение объектов, сегментация, классификация, трекинг);
- Работу в команде (управление проектами и задачами);
- Интеграцию с AI-моделями для автоматической разметки;
- Экспорт в форматы COCO, YOLO, Pascal VOC и др.;
- Работу с облачными хранилищами и 3D-данными.

#### 2.3. Библиотека аугментации Albumentations

**Albumentations** – это быстрая, гибкая и удобная библиотека для аугментации изображений, оптимизированная для задач компьютерного зрения. Библиотека содержит более 70 различных аугментаций. Ключевые особенности:
- Поддержка различных типов аннотаций: ограничивающие рамки (bounding boxes), ключевые точки (keypoints), маски сегментации;
- Возможность создания пайплайнов аугментаций с вероятностным применением;
- Высокая производительность;
- Сериализация пайплайнов в форматы JSON для воспроизводимости.

### 4. Порядок выполнения работы

#### 4.1. Разметка изображений с помощью VIA

**Цель:** освоить базовую разметку изображений с использованием лёгкого инструмента VIA.

1. **Скачайте VIA** с официального репозитория: [https://www.robots.ox.ac.uk/~vgg/software/via/](https://www.robots.ox.ac.uk/~vgg/software/via/). Достаточно скачать один HTML-файл (`via.html`), который запускается в браузере.

2. **Загрузите изображения** в VIA через меню `Image > Load or Add Images`. Выберите подготовленный набор изображений.

3. **Создайте атрибуты** для аннотаций (опционально). В меню `Attributes` можно определить атрибуты, например, для класса объекта (имя, тип).

4. **Выполните разметку**:
   - Для каждого объекта на изображении создайте **ограничивающую рамку** (Bounding Box) с помощью инструмента `Rectangle`.
   - Добавьте **полигональную разметку** (Polygon) для объектов со сложной формой.
   - Присвойте каждой аннотации атрибут класса (например, "cat", "dog").

5. **Экспортируйте аннотации** в формате JSON через меню `Annotation > Export annotations (as JSON)`. Сохраните файл.

6. **Изучите структуру** полученного JSON-файла. Обратите внимание на поля, описывающие координаты аннотаций и атрибуты.

#### 4.2. Разметка изображений с помощью CVAT

**Цель:** освоить профессиональный инструмент разметки для работы с большими проектами.

1. **Зарегистрируйтесь** на платформе [app.cvat.ai](https://app.cvat.ai) или установите CVAT локально (для продвинутых пользователей).

2. **Создайте новый проект** в CVAT. Настройте **лейблы** (классы объектов), которые будут использоваться при разметке.

3. **Создайте задачу (Task)** внутри проекта. Загрузите изображения для разметки. Настройте размер сегментов (frame) и укажите лейблы.

4. **Выполните разметку** с использованием различных инструментов:
   - **Rectangle** – для ограничивающих рамок.
   - **Polygon** – для сегментации объектов сложной формы.
   - **Points** – для разметки ключевых точек.

5. **Проверьте качество** разметки с помощью функции `Validation`. При необходимости исправьте ошибки.

6. **Экспортируйте** размеченный набор данных в одном из форматов: **COCO**, **YOLO**, **Pascal VOC**. Выберите формат, соответствующий задаче, для которой готовятся данные.

#### 4.3. Аугментация изображений и аннотаций с помощью Albumentations

**Цель:** освоить автоматическую аугментацию изображений с соответствующим преобразованием аннотаций.

1. **Установите Albumentations**:
   ```bash
   pip install albumentations
   ```

2. **Загрузите размеченные данные** из формата, полученного на предыдущих этапах. Для упрощения можно использовать разметку в формате COCO или собственный формат.

3. **Создайте пайплайн аугментаций**. Пример базового пайплайна для задачи обнаружения объектов:
   ```python
   import albumentations as A
   import cv2

   transform = A.Compose([
       A.RandomRotate90(p=0.5),
       A.Flip(p=0.5),
       A.RandomBrightnessContrast(p=0.2),
       A.GaussNoise(p=0.2),
   ], bbox_params=A.BboxParams(format='pascal_voc', label_fields=['labels']))
   ```

4. **Примените аугментации** к каждому изображению и его аннотациям. Для каждой аугментации визуализируйте результат с отображением преобразованных рамок или полигонов.
   ```python
   transformed = transform(image=image, bboxes=bboxes, labels=labels)
   transformed_image = transformed['image']
   transformed_bboxes = transformed['bboxes']
   ```

5. **Создайте несколько вариантов** аугментаций для каждого изображения, чтобы увеличить размер обучающей выборки в 5–10 раз.

6. **Сохраните аугментированный набор данных** в том же формате, что и исходный (или в формате, удобном для дальнейшего использования).

#### 4.4. Сравнительный анализ инструментов разметки

**Цель:** оценить удобство и эффективность различных инструментов разметки.

1. Сравните VIA и CVAT по следующим критериям:
   - Простота установки и запуска.
   - Скорость работы с набором из 10 изображений.
   - Функциональность (типы аннотаций, поддержка атрибутов).
   - Возможности командной работы.
   - Форматы экспорта.
   - Интеграция с другими инструментами.

2. Составьте таблицу сравнения и сделайте вывод о том, для каких задач какой инструмент более предпочтителен.

#### 4.5. Автоматизация процесса подготовки данных

**Цель:** разработать скрипт для автоматического преобразования размеченных данных в формат, пригодный для обучения модели.

1. Напишите скрипт на Python, который:
   - Загружает аннотации из JSON-файла (полученного из VIA или CVAT).
   - Преобразует их в формат, удобный для загрузки в Albumentations (список словарей с координатами и метками).
   - Применяет пайплайн аугментаций.
   - Сохраняет аугментированные изображения и обновлённые аннотации в целевой формат (например, COCO JSON или YOLO TXT).

2. Скрипт должен быть параметризуемым: количество аугментаций на изображение, типы аугментаций, формат ввода/вывода.
