### 2. Теоретические сведения (краткий конспект)
#### 2.1. Виды сегментации
- **Семантическая** – каждому пикселю присваивается класс объекта (без различения отдельных экземпляров).
- **Сегментация экземпляров** – выделяются отдельные объекты одного класса (каждый экземпляр имеет уникальную метку).
- **Паноптическая** – объединяет семантическую и экземплярную (все пиксели относятся либо к «вещи», либо к «фону»; вещи различаются по экземплярам).

#### 2.2. Метрики качества
- **IoU (Intersection over Union)** = |A ∩ B| / |A ∪ B|.
- **Коэффициент Дайса (Dice)** = 2|A ∩ B| / (|A| + |B|).
- **MPA (Mean Pixel Accuracy)** – средняя точность по классам: (1/C) Σ (TP_c / (TP_c + FP_c)).
- **MIoU (Mean IoU)** – среднее значение IoU по всем классам.
- **FWIoU (Frequency Weighted IoU)** – MIoU, взвешенное по частоте встречаемости классов.
- **mAP (mean Average Precision)** – используется для детекции объектов (в данной работе не обязательно).

#### 2.3. Обзор методов
- **Пороговые**: глобальный порог (Оцу), адаптивные (Ниблэк, Саувола) – хорошо для однородных объектов, чувствительны к шуму.
- **Кластеризация** (K‑Means) – группирует пиксели по цветовым признакам; число кластеров задаётся вручную.
- **Региональный рост** – начинается с «затравочных» пикселей и объединяет соседей по критерию схожести.
- **Разбиение и слияние** – рекурсивное разбиение квадродеревом, затем слияние однородных областей.
- **Водораздел** – обрабатывает градиентное изображение, разделяет соприкасающиеся объекты (требует маркеров).
- **Mean Shift** – немодальная кластеризация в пространстве признаков; хорошо для текстур.
- **Нормализованный разрез графа (Normalized Cuts)** – глобальная оптимизация на графе пикселей, учитывает сходство и сегментирует на основе собственных векторов.

Методы глубокого обучения (U‑Net, Mask R‑CNN) в работе **не реализуются**, но допускается их использование для сравнения (по желанию).

### 7. Рекомендации и справочные материалы

#### 7.1. Полезные функции OpenCV / scikit‑image
- `cv2.threshold` – глобальный порог (с Оцу).
- `cv2.adaptiveThreshold` – адаптивный порог (методы Гаусса и среднего).
- `cv2.kmeans` – кластеризация.
- `cv2.watershed` – водораздел (требует маркеры).
- `skimage.filters.threshold_niblack`, `threshold_sauvola`.
- `skimage.segmentation.watershed` – альтернативный водораздел.
- `skimage.segmentation.slic` – сверхпиксели (на основе Mean Shift).
- `skimage.segmentation.quickshift` – ещё один вариант.
- `skimage.measure.label` – маркировка связных областей.
- `skimage.metrics` – для вычисления метрик (например, `skimage.metrics.adapted_rand_error` – но для IoU и Dice лучше написать самим).

#### 7.2. Формулы для адаптивных методов
- **Ниблэк**: порог для пикселя (x,y) вычисляется как  
  `T(x,y) = m(x,y) + k * s(x,y)`,  
  где `m` – среднее в окне, `s` – стандартное отклонение, `k` – коэффициент (обычно -0.2 .. 0.2).
- **Саувола**:  
  `T(x,y) = m(x,y) * (1 + k * (s(x,y)/R - 1))`,  
  где `R` – максимальное отклонение (для 8-битных изображений R=128), `k` – обычно 0.2..0.5.

#### 7.3. Алгоритм регионального роста (псевдокод)
```
1. Выбрать множество затравочных пикселей (можно задать вручную или выбрать на основе гистограммы).
2. Для каждого семени создать пустой регион.
3. Использовать очередь (BFS):
   - Извлечь пиксель из очереди.
   - Для каждого соседа (4- или 8-связность):
        если сосед не посещён и разность яркости с текущим пикселем < порог:
            добавить в регион, поместить в очередь.
4. Получить маску регионов.
```

#### 7.4. Алгоритм разбиения и слияния (идея)
- Функция `split(region)`:
  - Если дисперсия яркости в регионе > заданного порога, разделить на 4 подрегиона (квадродерево).
  - Рекурсивно продолжать.
- После разбиения – функция `merge`:
  - Для соседних регионов, если разность их средних значений меньше порога, объединить.
- Результат – маска с уникальными метками для каждого финального региона.
