## 3. Теоретические сведения

### 3.1. Классы генеративных моделей

Генеративные модели решают задачу приближения распределения данных p(x) и синтеза новых объектов из этого распределения.

**Основные подходы к генерации изображений:**

| Класс | Принцип | Достоинства | Недостатки |
|-------|---------|-------------|------------|
| **VAE** (Variational Autoencoder) | Кодирование изображения в вероятностное латентное пространство и декодирование из него; обучение через максимизацию ELBO | Стабильное обучение, осмысленное латентное пространство | Размытость генераций |
| **GAN** (Generative Adversarial Network) | Состязательное обучение генератора и дискриминатора | Резкие, реалистичные изображения | Нестабильность обучения, mode collapse |
| **DM** (Diffusion Models) | Постепенное зашумление данных и обучение обратного процесса пошагового удаления шума | Наилучшее качество и разнообразие | Медленный семплинг (много шагов) |
| **Flow-based** | Обратимые преобразования с вычислимым якобианом; точное значение правдоподобия | Точная оценка p(x), обратимость | Ограничения на архитектуру, большие модели |

### 3.2. Ускорение семплинга диффузионных моделей

Классическая схема DDPM требует ~1000 шагов удаления шума. Применяются ускоренные планировщики (samplers/schedulers):

- **DDIM (Denoising Diffusion Implicit Models)** — детерминированный семплинг по неполной последовательности шагов (20–50 шагов вместо 1000);
- **PLMS (Pseudo Linear Multistep)** — псевдо-многошаговый численный метод, ускоряющий сходимость;
- **DPM-Solver / DPM-Solver++** — специализированный решатель ОДУ диффузионного процесса, дающий качественные изображения за 10–25 шагов.

### 3.3. Латентные диффузионные модели (LDM) и Stable Diffusion

Идея LDM — выполнять диффузию не в пространстве пикселей, а в компактном латентном пространстве, полученном предварительно обученным VAE. Это снижает вычислительные затраты на порядки. **Stable Diffusion** состоит из трёх компонентов:

1. **VAE** — сжимает изображение в латент и восстанавливает обратно;
2. **U-Net** — предсказывает шум в латентном пространстве, обусловливаясь на текстовый запрос через cross-attention;
3. **Текстовый энкодер (CLIP)** — преобразует текстовый запрос в эмбеддинги.

Ключевые параметры генерации: число шагов семплинга, guidance scale (сила следования тексту), seed.

### 3.4. Оценка качества сгенерированных изображений

- **FID (Fréchet Inception Distance)** — расстояние между статистиками признаков реальных и сгенерированных изображений; чем меньше, тем лучше;
- **IS (Inception Score)** — оценивает уверенность классификации и разнообразие; чем больше, тем лучше;
- **LPIPS** — перцептивное расстояние между парами изображений;
- **CLIP Score** — соответствие изображения текстовому запросу;
- **Экспертная (визуальная) оценка** — реалистичность, наличие артефактов, разнообразие.

### 3.5. Генерация видео

Основные подходы к генерации видео:

- **Пространственно-временная генерация** — 3D-свёртки/пространственно-временное внимание генерируют весь клип целиком (Video Diffusion Models);
- **Последовательная генерация кадров или латентных представлений** — кадры генерируются по одному с обусловливанием на предыдущие;
- **Авторегрессия на основе трансформеров** — видео как последовательность дискретных токенов (VideoGPT и др.);
- **Генерация в скрытом пространстве** — диффузия в латентном пространстве видео (Stable Video Diffusion, AnimateDiff).

Отдельное направление — **федеративное обучение диффузионных моделей**: модель обучается на распределённых данных без их централизованного сбора, что важно при работе с приватными изображениями.

**Задание 1.1.** Реализуйте и обучите VAE на Fashion-MNIST (или фрагменте CelebA):

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from torchvision import datasets, transforms

class VAE(nn.Module):
    def __init__(self, latent_dim=32):
        super().__init__()
        self.enc = nn.Sequential(
            nn.Linear(784, 400), nn.ReLU())
        self.fc_mu = nn.Linear(400, latent_dim)
        self.fc_logvar = nn.Linear(400, latent_dim)
        self.dec = nn.Sequential(
            nn.Linear(latent_dim, 400), nn.ReLU(),
            nn.Linear(400, 784), nn.Sigmoid())

    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        return mu + std * torch.randn_like(std)

    def forward(self, x):
        h = self.enc(x.view(-1, 784))
        mu, logvar = self.fc_mu(h), self.fc_logvar(h)
        z = self.reparameterize(mu, logvar)
        return self.dec(z), mu, logvar

def vae_loss(recon, x, mu, logvar):
    bce = F.binary_cross_entropy(recon, x.view(-1, 784), reduction='sum')
    kld = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return bce + kld
```
