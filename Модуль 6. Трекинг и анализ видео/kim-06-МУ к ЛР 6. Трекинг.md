## 3. Теоретические сведения

### 3.1. Трекинг объектов

Трекинг объектов — это динамический процесс в компьютерном зрении, включающий идентификацию конкретных сущностей на видео и мониторинг их перемещения по последовательности кадров. В отличие от детекции, которая выполняется на каждом кадре независимо, трекинг устанавливает связи между обнаруженными объектами на разных кадрах, присваивая им уникальные идентификаторы.

**Основные подходы к трекингу:**

- **Трекинг на основе детекции (Tracking-by-Detection):** Сначала на каждом кадре выполняется детекция объектов, затем результаты детекции связываются между кадрами с помощью алгоритмов ассоциации. К этому классу относятся DeepSORT, SORT, ByteTrack.

- **Трекинг без детекции:** Алгоритмы типа KLT, Mean-Shift, которые отслеживают объект на основе его визуальных характеристик без явной детекции на каждом кадре.

**Популярные алгоритмы трекинга:**

| Алгоритм | Особенности |
|----------|-------------|
| DeepSORT | Использует глубокие признаки для повторной идентификации объектов, устойчив к перекрытиям |
| ByteTrack | Высокая производительность, эффективно работает с низкокачественными детекциями |
| SORT | Простой и быстрый, использует фильтр Калмана и венгерский алгоритм |

### 3.2. Распознавание действий

Распознавание действий (Action Recognition) — задача определения типа действия, выполняемого человеком или объектом на видео. В отличие от классификации статических изображений, распознавание действий требует учёта временной динамики.

**Основные архитектуры:**

- **I3D (Inflated 3D Networks):** Использует 3D-свёртки для извлечения пространственно-временных признаков. Архитектура основана на "раздувании" 2D-свёрток в 3D, что позволяет использовать предварительно обученные на изображениях веса.

- **SlowFast:** Двухпоточная архитектура с "медленным" потоком для анализа статических сцен и "быстрым" потоком для захвата движения.

- **TSN (Temporal Segment Networks):** Сегментирует видео на части и агрегирует информацию из каждой части для классификации.

### 3.3. Комбинированные системы

Современные системы трекинга и распознавания действий часто строятся по принципу "конвейера":

1. **Детектор** (YOLO, MobileNet SSD) обнаруживает объекты на каждом кадре.
2. **Трекер** (DeepSORT, ByteTrack) связывает обнаружения между кадрами.
3. **Классификатор действий** (SlowFast, I3D) анализирует траектории и определяет действия.

```python
from ultralytics import YOLO

model = YOLO('yolov8n.pt')
cap = cv2.VideoCapture('video.mp4')

while cap.isOpened():
    ret, frame = cap.read()
    if not ret:
        break
    
    # Включение трекинга
    results = model.track(frame, persist=True)
    
    # Визуализация с ID объектов
    for box in results[0].boxes:
        x1, y1, x2, y2 = map(int, box.xyxy[0])
        track_id = int(box.id[0]) if box.id is not None else -1
        cls = int(box.cls[0])
        conf = float(box.conf[0])
        
        cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 255, 0), 2)
        cv2.putText(frame, f'ID:{track_id} {model.names[cls]}',
                    (x1, y1-10), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0,255,0), 2)
    
    cv2.imshow('Tracking', frame)
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```
