### 4. Рекомендации по проведению отдельных разделов

#### 4.1. Архитектуры нейросетей для контуров

**Ключевые архитектуры для рассмотрения:**

1. **MLP (Многослойный перцептрон):**
   - Вход: вектор комплексных чисел, преобразованный в вектор действительных чисел `[Re(z_0), Im(z_0), ..., Re(z_{n-1}), Im(z_{n-1})]`.
   - Подходит для малых размерностей (n ≤ 64).
   - Преимущество: простота реализации.

2. **CNN на основе 1D свёрток:**
   - Вход: тензор формы `[2, n]` (действительная и мнимая части как два канала).
   - Преимущество: выявление локальных паттернов в последовательности.
   - Хорошо работает для любых размерностей.

3. **RNN/LSTM (рекуррентные сети):**
   - Вход: последовательность комплексных чисел.
   - Преимущество: обработка циклической природы контуров.
   - Требует больше данных для обучения.

4. **Архитектура с инвариантностью к циклическому сдвигу:**
   - Использование свёрточных слоёв с `padding='circular'`.
   - Добавление слоя глобального пулинга для инвариантности.

**Рекомендация:** Предложить студентам реализовать как минимум MLP и 1D-CNN, сравнить их эффективность.

#### 4.2. Генерация данных

**Ключевые моменты:**
- Генерация должна быть полностью случайной, чтобы сеть не запоминала конкретные примеры.
- Для каждого класса должно быть одинаковое количество примеров (баланс классов).
- Шум следует добавлять к абсолютным координатам, а не к цепному коду.
- Сохраняйте истинные параметры преобразований для возможности оценки точности.

**Рекомендация:** Реализовать генератор как класс с методами `generate_batch(batch_size)` для удобства использования в обучении.

#### 4.3. Обучение нейросети

**Ключевые моменты:**
- Использовать кросс-энтропийную функцию потерь (CrossEntropyLoss).
- Выбрать оптимизатор: Adam с начальной скоростью обучения 0.001.
- Реализовать раннюю остановку (early stopping) по метрике на валидационной выборке.
- Использовать пакетную обработку (batch size 32–128 в зависимости от объёма данных).
- Применять регуляризацию: Dropout (0.3–0.5), L2-регуляризация (weight decay).

**Типичные ошибки:**
- Слишком большой learning rate → расходимость.
- Слишком маленькая сеть → недообучение.
- Слишком большая сеть при малом объёме данных → переобучение.
- Неправильная нормализация входных данных (нужно нормализовать по среднему и дисперсии).

**Рекомендация:** Показать студентам, как использовать TensorBoard или WandB для мониторинга обучения.

#### 4.4. Оценка качества

- Матрица ошибок (confusion matrix) для визуализации ошибок между классами.
- Precision, Recall, F1-score для каждого класса.
- ROC-кривая и AUC для бинарных сценариев.

### 5. Типичные ошибки и способы их предотвращения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Сеть не обучается (loss не падает) | Неправильная нормализация данных | Нормализовать входные данные (mean=0, std=1) |
| Переобучение (train accuracy >> val accuracy) | Недостаточно регуляризации | Добавить Dropout, L2, увеличить объём данных |
| Модель не различает классы при повороте | Отсутствие инвариантности к повороту | Добавить аугментацию поворотами в обучающую выборку |
| Медленное обучение | Неэффективная загрузка данных | Использовать DataLoader с несколькими рабочими процессами |
| Нестабильное обучение | Слишком высокий learning rate | Использовать learning rate scheduling |
| Ошибки на GPU (CUDA out of memory) | Слишком большой batch size | Уменьшить batch size или использовать градиентное накопление |
| Низкая точность при большом n | Сеть недостаточно глубокая | Увеличить число слоёв или использовать LSTM |
| Плохая работа на реальных изображениях | Данные обучения слишком идеальные | Добавить больше шума и искажений в обучающую выборку |

### 8. Контрольные вопросы для защиты (с ответами для преподавателя)

1. **Почему для обработки контуров лучше использовать 1D-свёртки, а не 2D?**  
   *Ответ:* Контур является последовательностью точек, упорядоченных вдоль границы объекта. Это одномерная структура, поэтому 1D-свёртки естественным образом обрабатывают такие данные. 2D-свёртки предназначены для пространственных данных (изображений) и не учитывают последовательный характер контура.

2. **Как обеспечить инвариантность нейросети к циклическому сдвигу контура?**  
   *Ответ:* Использовать циклическую аугментацию данных (включать случайные сдвиги при генерации выборки) или использовать архитектуру с глобальным пулингом (max/avg pooling), который агрегирует информацию по всей длине последовательности, делая её инвариантной к сдвигу.

3. **В чём преимущества нейросетевого классификатора перед классическим (максимального правдоподобия)?**  
   *Ответ:* Нейросети могут обучаться на большом объёме данных и выявлять сложные нелинейные зависимости. Они более устойчивы к шуму и искажениям. Также они могут автоматически выделять признаки, тогда как классический классификатор использует только скалярное произведение.

4. **Какие архитектуры подходят для обработки последовательностей комплексных чисел?**  
   *Ответ:* MLP (после разделения на действительную и мнимую части), 1D-CNN с двумя входными каналами (Re и Im), LSTM/GRU, а также специализированные комплексные нейросети (Complex-Valued Neural Networks).

5. **Как аугментация данных влияет на качество классификации?**  
   *Ответ:* Аугментация увеличивает разнообразие обучающей выборки, что помогает сети обобщать и делает её устойчивой к различным искажениям (поворот, масштаб, шум). Это особенно важно при ограниченном количестве исходных эталонов.

6. **Что такое переобучение и как его предотвратить?**  
   *Ответ:* Переобучение — это ситуация, когда модель запоминает обучающую выборку, но плохо работает на новых данных. Предотвращается регуляризацией (Dropout, L2), ранней остановкой, увеличением объёма данных, аугментацией.

7. **Почему при использовании нейросетей важно нормализовать входные данные?**  
   *Ответ:* Нормализация ускоряет сходимость градиентного спуска, улучшает численную стабильность и позволяет использовать стандартные значения параметров инициализации. Особенно важна для глубоких сетей с множеством слоёв.

8. **Как выбрать оптимальный размер батча при обучении?**  
   *Ответ:* Размер батча выбирается из компромисса: меньший батч даёт более шумные градиенты и лучшее обобщение, но медленнее обучает; больший батч быстрее использует GPU, но может ухудшить обобщение. Типичный диапазон: 32–128.

9. **В чём отличие обучения нейросети с нуля от fine-tuning предобученной модели?**  
   *Ответ:* Обучение с нуля требует большого объёма данных и вычислительных ресурсов. Fine-tuning использует предобученные веса (например, на больших наборах данных) и дообучает на задаче, что требует меньше данных и времени.

10. **Каким образом можно интерпретировать работу нейросети для распознавания контуров?**  
    *Ответ:* Использовать методы объяснимого AI: Grad-CAM для визуализации важных участков контура, анализ активаций нейронов, визуализация фильтров первого слоя. Это помогает понять, какие признаки сеть выделяет.
