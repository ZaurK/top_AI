### 2. Теоретические сведения

#### 2.1. Нейросетевой подход к распознаванию контуров

В отличие от классического подхода, основанного на скалярном произведении и корреляции, нейросетевой подход позволяет обучить модель на большом количестве примеров и выявлять сложные нелинейные зависимости.

**Преимущества нейросетевого подхода:**
- Устойчивость к шуму и искажениям.
- Автоматическое выделение признаков.
- Возможность работы с большими наборами классов.
- Адаптивность к различным типам искажений.

**Недостатки:**
- Требует большого объёма данных для обучения.
- Сложность интерпретации результатов.
- Большие вычислительные затраты на обучение.

#### 2.2. Представление данных для нейросети

Контур задаётся последовательностью комплексных чисел:
\[
\mathbf{s} = \{s_0, s_1, \ldots, s_{n-1}\}, \quad s_k \in \mathbb{C}
\]

Для подачи в нейросеть комплексный вектор преобразуется в действительный:
\[
\mathbf{x} = [\text{Re}(s_0), \text{Im}(s_0), \ldots, \text{Re}(s_{n-1}), \text{Im}(s_{n-1})]
\]
или представляется как тензор формы `[2, n]` (два канала: действительная и мнимая части).

#### 2.3. Архитектуры нейросетей для контуров

**2.3.1. MLP (Многослойный перцептрон)**

Простейшая архитектура, состоит из полносвязных слоёв:

```
Input (2n) → Dense(128) → ReLU → Dense(64) → ReLU → Dense(M) → Softmax
```

**2.3.2. 1D-CNN (Свёрточная нейронная сеть)**

Эффективно выявляет локальные паттерны в последовательности:

```
Input [2, n] → Conv1D(32, kernel=5, padding=circular) → ReLU → MaxPool1D(2)
→ Conv1D(64, kernel=3, padding=circular) → ReLU → MaxPool1D(2)
→ GlobalAvgPool1D → Dense(64) → ReLU → Dense(M) → Softmax
```

**2.3.3. LSTM (Рекуррентная сеть)**

Учитывает последовательный характер контура:

```
Input [n, 2] → LSTM(64, bidirectional=True) → GlobalAvgPool1D → Dense(M) → Softmax
```

#### 2.4. Генерация синтетических данных

Для обучения нейросети требуется большое количество примеров. Из каждого эталонного контура генерируются вариации со случайными преобразованиями:

1. **Циклический сдвиг** `d ∈ [0, n-1]`:
   \[
   \mathbf{s}^{(d)} = \{s_d, s_{d+1}, \ldots, s_{n-1}, s_0, \ldots, s_{d-1}\}
   \]

2. **Масштабирование** `a ∈ [0.5, 2.0]`:
   \[
   s'_k = a \cdot s_k
   \]

3. **Поворот** `α ∈ [0, 2π)`:
   \[
   s'_k = s_k \cdot e^{j\alpha}
   \]

4. **Шум** `σ ∈ [0, 0.3]` (добавляется к абсолютным координатам):
   \[
   x'_k = x_k + \eta_k, \quad \eta_k \sim \mathcal{N}(0, \sigma^2)
   \]

#### 2.5. Обучение нейросети

**Функция потерь:** Кросс-энтропия (CrossEntropyLoss)

**Оптимизатор:** Adam с learning rate 0.001

**Регуляризация:**
- Dropout (0.3–0.5)
- L2-регуляризация (weight decay = 1e-4)
- Ранняя остановка (early stopping)

**Метрики:**
- Accuracy
- Матрица ошибок (Confusion Matrix)
- Precision, Recall, F1-score

### 3. Порядок выполнения работы

#### 3.1. Разработка программных функций

Реализуйте на языке Python с использованием PyTorch (или TensorFlow/Keras) следующие функции:

1. **Генератор данных `ContourDataset`**:
   - Загружает эталонные контуры из банка.
   - Генерирует синтетические примеры с преобразованиями.
   - Разделяет данные на train/val/test.

2. **Класс `ContourMLP`** — многослойный перцептрон.

3. **Класс `ContourCNN1D`** — свёрточная сеть с одномерными свёртками.

4. **Класс `ContourLSTM`** — рекуррентная сеть.

5. **Функция `train_model(model, train_loader, val_loader, epochs, lr)`**:
   - Реализует цикл обучения.
   - Сохраняет лучшую модель по валидационной точности.
   - Выводит графики потерь и точности.

6. **Функция `evaluate_model(model, test_loader)`**:
   - Оценивает модель на тестовой выборке.
   - Вычисляет accuracy, строит матрицу ошибок.

7. **Сравнение с классическим классификатором**:
   - Использует классификатор из предыдущей работы.

#### 3.2. Генерация данных

1. Загрузите банк эталонных контуров из JSON-файла.
2. Для каждого эталона сгенерируйте обучающую выборку:
   - 1000 примеров на класс (можно варьировать).
   - Используйте случайные преобразования: сдвиг `d`, масштаб `a`, поворот `α`, шум `σ`.
3. Разделите данные на:
   - Train: 70%
   - Validation: 15%
   - Test: 15%
4. Нормализуйте данные по среднему и стандартному отклонению.

#### 3.3. Обучение нейросети

1. Выберите одну из архитектур (MLP, CNN, LSTM) и реализуйте её.
2. Обучите модель, контролируя точность на валидационной выборке.
3. Используйте раннюю остановку (если точность на val не улучшается 10 эпох).
4. Визуализируйте графики потерь и точности.

#### 3.4. Оценка качества

1. Оцените точность модели на тестовой выборке.
2. Постройте матрицу ошибок.
3. Вычислите precision, recall, f1-score для каждого класса.
4. Проанализируйте, какие классы сеть путает чаще всего.

#### 3.5. Сравнение с классическим классификатором

1. Используйте классификатор из предыдущей работы (максимального правдоподобия).
2. Оцените его точность на тех же тестовых данных.
3. Сравните результаты в таблице.
4. Сделайте выводы о преимуществах и недостатках каждого подхода.

#### 3.6. Исследование устойчивости

1. Протестируйте обученную модель на данных с разным уровнем шума:
   - σ = 0.01, 0.05, 0.1, 0.2, 0.3
2. Постройте график зависимости точности от уровня шума.
3. Сравните с классическим классификатором.
