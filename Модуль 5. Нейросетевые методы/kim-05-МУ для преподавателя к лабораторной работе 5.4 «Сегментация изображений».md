### 4. Рекомендации по проведению отдельных разделов

#### 4.1. Вступительная лекция

**Ключевые темы:**
1. **Семантическая vs Инстансная сегментация:** Показать примеры.
2. **Архитектура U-Net:**
   - Encoder (сжатие) — уменьшение размерности.
   - Decoder (расширение) — восстановление размерности.
   - Skip-connections — передача детальной информации.
3. **Транспонированная свёртка (Up-conv):** Как она работает.
4. **Функции потерь:** Categorical Cross-Entropy, Dice Loss.
5. **Метрики:** IoU, mIoU, Pixel Accuracy.

**Визуальный материал:** Схема U-Net с пояснением, почему skip-connections так важны.

#### 4.2. Реализация U-Net

**Ключевые моменты для объяснения:**
- Каждый блок encoder содержит два свёрточных слоя и один pooling.
- Каждый блок decoder содержит up-sampling, конкатенацию с skip-connection и два свёрточных слоя.
- Размеры должны совпадать для конкатенации.

**Что происходит при конкатенации (concatenate)?**
- Соединяются карты признаков с encoder и decoder.
- Это позволяет сохранить детальную информацию о границах объектов.

**Типичные ошибки:**
- Неправильные размеры для конкатенации.
- Использование conv2d вместо conv2dtranspose.
- Забывают про `padding='same'`.

#### 4.3. Подготовка данных

**Важные аспекты:**
- Маски должны быть целочисленными (индексы классов).
- Для обучения использовать `categorical_crossentropy` и one-hot маски.
- При изменении размера маски использовать `interpolation=INTER_NEAREST`.

```python
# Правильный способ изменения размера маски
mask = cv2.resize(mask, (128, 128), interpolation=cv2.INTER_NEAREST)
```

**Обсуждение:** Почему для масок нельзя использовать обычную интерполяцию (билинейную)? (Появляются промежуточные значения, не являющиеся классами).

#### 4.4. Обучение

**Колбэки:** EarlyStopping, ModelCheckpoint, ReduceLROnPlateau.

**На что обратить внимание:**
- При несбалансированных классах (мало объектов фона) можно использовать веса классов.
- Dice Loss помогает при дисбалансе классов.

```python
# Пример Dice Loss
def dice_loss(y_true, y_pred, smooth=1e-6):
    intersection = K.sum(y_true * y_pred, axis=[1,2,3])
    union = K.sum(y_true, axis=[1,2,3]) + K.sum(y_pred, axis=[1,2,3])
    dice = (2. * intersection + smooth) / (union + smooth)
    return 1 - dice
```

**Важно:** Отображать три изображения рядом:
1. Исходное изображение.
2. Истинная маска (ground truth).
3. Предсказанная маска.

Это позволяет визуально оценить качество сегментации.

### 5. Типичные ошибки

| Ошибка | Решение |
|--------|---------|
| Маски имеют значения 0–255 вместо классов | Использовать `cv2.IMREAD_GRAYSCALE` и нормализовать |
| Неправильные размеры при конкатенации | Проверить `padding='same'` во всех свёртках |
| Потери не уменьшаются | Проверить формат масок (one-hot) |
| Границы объектов размыты | Увеличить количество слоёв в decoder |
| Модель предсказывает только фон | Добавить веса классов или использовать Dice Loss |

### 6. Контрольные вопросы

1. **В чём отличие семантической сегментации от инстансной?**  
   *Ответ:* Семантическая классифицирует каждый пиксель по классу; инстансная — выделяет отдельные объекты.

2. **Какую роль играют skip-connections в U-Net?**  
   *Ответ:* Передают детальную информацию с encoder на decoder, позволяя восстановить точные границы объектов.

3. **Что такое транспонированная свёртка?**  
   *Ответ:* Операция, увеличивающая пространственную размерность, обратная обычной свёртке.

4. **Как вычисляется IoU?**  
   *Ответ:* \( IoU = \frac{TP}{TP+FP+FN} \) — отношение пересечения к объединению.

5. **Почему маски нужно изменять с `INTER_NEAREST`?**  
   *Ответ:* Чтобы не создавать промежуточных значений, не являющихся классами.
