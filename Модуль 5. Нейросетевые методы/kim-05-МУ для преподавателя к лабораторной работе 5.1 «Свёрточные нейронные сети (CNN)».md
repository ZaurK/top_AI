### 4. Рекомендации по проведению отдельных разделов

#### 4.1. Вступительная лекция

**Ключевые темы для объяснения:**
1. **Почему обычные нейронные сети не подходят для изображений?**
   - Слишком много параметров.
   - Отсутствие инвариантности к сдвигу.
   - Потеря пространственной структуры.

2. **Как работает свёртка?**
   - Демонстрация на анимации.
   - Формула: \( (I * K)(i,j) = \sum_{m}\sum_{n} I(i+m, j+n) \cdot K(m,n) \).
   - Понятие ядра (фильтра), карты признаков.

3. **ReLU и другие функции активации.**
   - Преимущества ReLU перед сигмоидой.
   - Проблема «умирающих» нейронов.

4. **Подвыборка (Pooling).**
   - Max pooling vs Average pooling.
   - Уменьшение размерности, увеличение receptive field.

5. **Регуляризация.**
   - Dropout: случайное отключение нейронов.
   - Batch Normalization: нормализация активаций.
   - L1/L2 регуляризация.

6. **Функции потерь и оптимизаторы.**
   - Cross-entropy для классификации.
   - Adam, SGD с моментом.

**Визуальный материал:** Подготовьте слайды с анимациями свёртки, pooling и схемами архитектур.

#### 4.2. Разработка моделей

**Простая модель (базовый уровень):**
```python
model = Sequential([
    Conv2D(32, (3,3), activation='relu', input_shape=(32,32,3)),
    MaxPooling2D((2,2)),
    Conv2D(64, (3,3), activation='relu'),
    MaxPooling2D((2,2)),
    Conv2D(64, (3,3), activation='relu'),
    Flatten(),
    Dense(64, activation='relu'),
    Dense(10, activation='softmax')
])
```

**Обсуждение:** Почему именно такие размеры? Что будет, если убрать pooling?

**Глубокая модель с регуляризацией:**
```python
model = Sequential([
    Conv2D(32, (3,3), activation='relu', input_shape=(32,32,3)),
    BatchNormalization(),
    MaxPooling2D((2,2)),
    Conv2D(64, (3,3), activation='relu'),
    BatchNormalization(),
    MaxPooling2D((2,2)),
    Conv2D(128, (3,3), activation='relu'),
    BatchNormalization(),
    MaxPooling2D((2,2)),
    Flatten(),
    Dropout(0.5),
    Dense(256, activation='relu'),
    Dropout(0.3),
    Dense(10, activation='softmax')
])
```

**Ключевые вопросы:**
- Зачем добавлять BatchNormalization после каждого свёрточного слоя?
- Почему Dropout применяется только к полносвязным слоям?

#### 4.3. Обучение и мониторинг

**Колбэки, которые обязательно показать:**
- `EarlyStopping` — предотвращает переобучение.
- `ModelCheckpoint` — сохраняет лучшую модель.
- `ReduceLROnPlateau` — снижает скорость обучения при затухании градиента.

**Как интерпретировать графики:**
- Training loss ↓ и Validation loss ↓ — хорошее обучение.
- Training loss ↓, Validation loss ↑ — переобучение.
- Training loss ↓, Validation loss стабильна — возможно, модель слишком простая (underfitting).

**Типичные ошибки:**
- Слишком большая скорость обучения — loss не сходится.
- Слишком маленькая — обучение слишком медленное.
- Отсутствие нормализации данных — градиенты взрываются.
- Неправильный размер входных данных.

#### 4.4. Аугментация данных

**Обоснование:** Аугментация увеличивает эффективный размер выборки и делает модель устойчивее к вариациям.

**Параметры для обсуждения:**
- `rotation_range`: поворот до 20° — разумный выбор для CIFAR-10.
- `width_shift_range` и `height_shift_range`: сдвиги до 20%.
- `horizontal_flip`: допустимо, если объекты не имеют строгой ориентации.
- `zoom_range`: небольшое масштабирование.

**Важно:** Аугментация применяется только к обучающей выборке, не к валидационной и тестовой.

### 5. Типичные ошибки и способы их предотвращения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Модель не обучается (loss не падает) | Неправильная нормализация данных | Разделить пиксели на 255.0 |
| Loss становится NaN | Слишком большая скорость обучения | Уменьшить learning_rate |
| Быстрое переобучение | Мало данных, нет регуляризации | Добавить Dropout, аугментацию |
| Медленное обучение | Нет BatchNormalization | Добавить BN после свёрточных слоёв |
| Неправильные размеры тензоров | Ошибка в параметрах слоёв | Проверить `input_shape` и размерности |
| Точность на валидации ниже случайной | Перепутаны метки | Проверить `to_categorical` |
| Не хватает памяти GPU | Слишком большой батч | Уменьшить batch_size |
| Модель не улучшается | Слишком глубокая для данных | Упростить архитектуру |

### 8. Контрольные вопросы для защиты (с ответами для преподавателя)

1. **Что такое операция свёртки и чем она отличается от полносвязного слоя?**  
   *Ответ:* Свёртка использует локальные связи и общие веса (ядра), что значительно уменьшает количество параметров и обеспечивает инвариантность к сдвигу. Полносвязный слой соединяет каждый нейрон со всеми нейронами предыдущего слоя.

2. **Почему ReLU лучше сигмоиды в скрытых слоях CNN?**  
   *Ответ:* ReLU не насыщается, производная равна 1 для положительных значений, что решает проблему затухающих градиентов. Также ReLU быстрее вычисляется.

3. **Что такое Max Pooling и зачем он нужен?**  
   *Ответ:* Max Pooling выбирает максимальное значение в окне. Он уменьшает пространственную размерность, увеличивает receptive field, делает модель устойчивее к малым смещениям и уменьшает количество параметров.

4. **Как работает Dropout и почему он помогает бороться с переобучением?**  
   *Ответ:* Dropout случайно «выключает» нейроны во время обучения, что предотвращает коадаптацию нейронов и заставляет модель изучать более устойчивые признаки.

5. **Что такое Batch Normalization?**  
   *Ответ:* BN нормализует активации внутри батча, центрируя и масштабируя их. Это ускоряет обучение, позволяет использовать бóльшие скорости обучения и действует как регуляризатор.

6. **Как выбрать оптимальную архитектуру CNN для конкретной задачи?**  
   *Ответ:* Исходить из сложности данных (размер изображения, количество классов). Начинать с простой модели, постепенно усложнять. Использовать transfer learning при недостатке данных.

7. **Что такое аугментация данных и зачем она нужна?**  
   *Ответ:* Аугментация — преобразование изображений (повороты, сдвиги, отражения), увеличивающее эффективный размер выборки и делающее модель устойчивее к вариациям.

8. **Как интерпретировать матрицу ошибок?**  
   *Ответ:* Матрица показывает, какие классы модель путает. Анализ позволяет понять слабые места модели и, возможно, добавить больше данных для проблемных классов.

9. **Что такое Transfer Learning?**  
   *Ответ:* Использование предобученной на большом наборе данных модели для новой задачи. Позволяет достичь хороших результатов с малым количеством данных.

10. **Как влияет размер батча на обучение?**  
    *Ответ:* Малый батч даёт более шумные градиенты (лучше обобщение, но медленнее сходимость). Большой батч даёт стабильные градиенты (быстрее сходимость, но хуже обобщение и больше памяти).
