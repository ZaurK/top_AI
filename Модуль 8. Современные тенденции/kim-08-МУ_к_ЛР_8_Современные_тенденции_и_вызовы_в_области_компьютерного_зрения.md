## 3. Теоретические сведения
### 3.1. Тенденции развития компьютерного зрения

До середины 2010-х годов компьютерное зрение развивалось как набор узкоспециализированных решений: отдельная модель под классификацию, отдельная — под детекцию, отдельная — под сегментацию, каждая обучалась на своём датасете под свой набор классов. Появление трансформерной архитектуры (Vision Transformer, ViT) и масштабных датасетов изображение-текст изменило парадигму: возникли модели, обучаемые на сотнях миллионов и миллиардах пар «изображение-текст» или немаркированных изображений, которые переносят полученные представления на десятки последующих задач без переобучения либо с минимальной дообучающей выборкой (few-shot/zero-shot).

Ключевые тенденции:

- Переход от свёрточных сетей (CNN) к трансформерным архитектурам (ViT) и гибридным решениям.
- Рост масштаба предобучающих данных и переход к слабо размеченным или неразмеченным выборкам (self-supervised, weakly-supervised обучение).
- Мультимодальность: совместное обучение на изображениях и естественном языке (CLIP-подобные модели).
- Открытый словарь (open-vocabulary): отказ от фиксированного набора классов в пользу произвольных текстовых запросов.
- Промптируемость (promptable models): модель принимает на вход не только изображение, но и подсказку — точку, бокс, маску, текст (SAM, Grounding DINO).
- Универсализация: одна модель решает множество задач, либо служит единым бэкбоном-признаковым экстрактором для множества последующих голов (AM-RADIO).
- Рост требований к интерпретируемости и надёжности решений по мере внедрения в критичные области (медицина, автономный транспорт, промышленность).

### 3.2. Задачи предобучения фундаментальных моделей CV

Под предобучением фундаментальной модели понимается обучение на вспомогательной (pretext) задаче на большом объёме данных с целью получить универсальные визуальные представления, пригодные для переноса на широкий спектр последующих (downstream) задач.

- **Контрастивное обучение изображение-текст (CLIP)**: модель обучается сближать эмбеддинги изображения и его текстового описания и отдалять эмбеддинги несоответствующих пар в общем векторном пространстве — естественный механизм zero-shot классификации и поиска.
- **Маскированное автокодирование (MAE)**: часть патчей изображения маскируется, модель обучается восстанавливать их по видимым фрагментам — аналог masked language modeling, перенесённый на изображения.
- **Самодистилляция без учителя (DINO, DINOv2)**: «студенческая» и «учительская» сети (учитель — EMA весов студента) обучаются давать согласованные представления на разных аугментациях одного изображения, без меток.
- **Промптируемая сегментация (SAM)**: модель обучается на масштабном датасете масок (SA-1B) выдавать маску объекта по произвольной пространственной подсказке (точка, бокс, грубая маска).
- **Совмещение детекции/подписей с языковыми моделями**: обучение на парах «изображение — текстовое описание региона» связывает текстовые фразы с объектами (Grounding DINO) либо генерирует связное описание изображения (image captioning).
- **Регрессия плотных карт (глубина, нормали)**: обучение на смеси синтетических данных с точной разметкой и реальных данных с псевдоразметкой от учителя обеспечивает устойчивую монокулярную оценку глубины в разных доменах (Depth Anything v2).

### 3.3. Универсальные визуальные модели: агломерация признаков (AM-RADIO)

Отдельные фундаментальные модели (CLIP, DINOv2, SAM и др.) обучены на разных задачах и обладают разными сильными сторонами: CLIP хорошо связывает изображение с семантикой текста, DINOv2 даёт устойчивые плотные признаки для сегментации и сопоставления, SAM специализируется на границах объектов. Идея агломерации (AM-RADIO — Agglomerative Models) заключается в дистилляции знаний сразу нескольких моделей-учителей в единый визуальный бэкбон-«студент»: он обучается одновременно воспроизводить признаки всех учителей на общем наборе изображений. В результате единая модель отдаёт признаки, пригодные для широкого круга последующих задач, заменяя собой несколько отдельных энкодеров и снижая вычислительные затраты при инференсе.

### 3.4. Обзор передовых готовых моделей компьютерного зрения

| Модель / система | Решаемая задача | Ключевая особенность |
|---|---|---|
| AM-RADIO | Агломерация признаков из нескольких учителей (CLIP, DINOv2, SAM и др.) | Единый универсальный бэкбон вместо нескольких специализированных сетей |
| Recognize Anything (RAM) | Распознавание тегов/сущностей без явных запросов | Мультитегирование открытого словаря, обучение на слабо размеченных парах |
| Image Captioning (BLIP-2, GIT и др.) | Генерация текстового описания изображения | Связка визуального энкодера и LLM через лёгкий адаптер |
| Grounding DINO | Детекция объектов по произвольному тексту (open-vocabulary) | Совмещает DINO-детектор с текстовым энкодером |
| SAM / SAM 2 | Сегментация «всего» по точке/боксу/маске (promptable) | Обучена на 1+ млрд масок (SA-1B) |
| MAE | Самообучение через восстановление замаскированных патчей | Простая и масштабируемая парадигма предобучения ViT |
| DINOv2 | Самообучение без учителя (self-distillation) | Признаки переносятся на детекцию, сегментацию, глубину без дообучения |
| CLIP | Контрастное сопоставление изображений и текста | Единое эмбеддинг-пространство «картинка-текст» |
| YOLO-World / YOLOE | Детекция открытого словаря в реальном времени | Скорость YOLO + текстовые промпты |
| Depth Anything v2 | Монокулярная карта глубины «всего» | Смесь синтетических и псевдоразмеченных реальных данных |
| SAMURAI | Zero-shot трекинг объектов в видео | Расширяет SAM 2 механизмом движения/памяти |

Grounding DINO ориентирован на максимальную точность и гибкость запроса, тогда как YOLO-World/YOLOE — на скорость инференса в реальном времени. SAMURAI расширяет SAM 2 механизмом отслеживания движения и состояния объекта во времени, что позволяет выполнять устойчивый zero-shot трекинг по одной начальной подсказке без дообучения под конкретную сцену.

### 3.5. Большое объединение: от универсальных моделей к гибридному человеко-машинному интеллекту

1. **Универсальные глубокие модели** — единичные нейросетевые архитектуры (ViT, ResNet-подобные бэкбоны), обобщающие решение одной категории задач на широкий круг доменов и классов за счёт масштабного предобучения.
2. **Универсальные модели** — объединение нескольких таких моделей и модальностей (зрение + язык) в единые системы, решающие множество задач через общий интерфейс промптов (текст, точка, бокс, маска), например AM-RADIO, CLIP-подобные и промптируемые модели.
3. **Универсальный гибридный искусственный интеллект** — объединение универсальных визуальных моделей с крупными языковыми моделями (LLM) и другими модальностями (речь, действия, планирование) в мультимодальные агентные системы, способные рассуждать об увиденном и планировать действия.
4. **Универсальный гибридный человеко-машинный интеллект** — следующий уровень, при котором мультимодальные ИИ-системы встраиваются в контур совместной работы с человеком, дополняя и усиливая его когнитивные и перцептивные возможности.

Эта последовательность отражает вектор развития области: от узких инструментов — к универсальным перцептивным моделям, далее — к мультимодальным рассуждающим системам, и в конечном счёте — к системам, спроектированным для эффективного взаимодействия человека и машины.

### 3.6. Интерпретируемость и объяснимость моделей в компьютерном зрении

- **Grad-CAM и модификации**: тепловая карта значимости областей изображения для предсказания класса на основе градиентов относительно карт признаков свёрточных слоёв.
- **Визуализация карт внимания** трансформерных архитектур (ViT, DINO, DINOv2) — на каких патчах «фокусируется» модель.
- **Визуализация эмбеддингов**: проекция (t-SNE, UMAP) многомерных представлений (например, CLIP) в 2D для оценки кластеризации классов.
- **Пробинг (probing)**: обучение линейных классификаторов поверх замороженных признаков — оценка линейной разделимости информации.
- **Анализ через промпты**: для SAM/Grounding DINO варьирование подсказки и наблюдение за изменением результата — интерактивный способ понять поведение модели.

Интерпретируемость особенно важна при внедрении фундаментальных моделей в чувствительные области: она позволяет выявлять смещения (bias) в данных, диагностировать ошибки и повышать доверие к автоматизированным решениям.

```
from transformers import CLIPProcessor, CLIPModel

clip_model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
clip_processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")
clip_model.eval()

# TODO: замените на свои гипотезы под конкретное изображение
text_labels = [
    "фотография улицы города",
    "фотография интерьера помещения",
    "фотография природы и леса",
    "промышленный объект или завод",
    "человек в защитной каске",
    "автомобиль на дороге",
]

def clip_zero_shot(image_path, labels):
    image = Image.open(image_path).convert("RGB")
    inputs = clip_processor(text=labels, images=image, return_tensors="pt", padding=True)
    with torch.no_grad():
        outputs = clip_model(**inputs)
    probs = outputs.logits_per_image.softmax(dim=1)[0].numpy()
    order = np.argsort(-probs)[:3]
    plt.figure(figsize=(5, 5))
    plt.imshow(image)
    plt.axis("off")
    title = "\n".join(f"{labels[i]}: {probs[i]:.2%}" for i in order)
    plt.title(title, fontsize=10)
    plt.show()
    return [(labels[i], float(probs[i])) for i in order]

for p in image_paths:
    print(p, "->", clip_zero_shot(p, text_labels))
```

import torch.nn.functional as F

def clip_attention_map(image_path):
    image = Image.open(image_path).convert("RGB")
    inputs = clip_processor(images=image, return_tensors="pt")
    with torch.no_grad():
        vision_outputs = clip_model.vision_model(
            pixel_values=inputs["pixel_values"], output_attentions=True
        )
    # внимание последнего слоя, усреднённое по головам, CLS-токен к патчам
    attn = vision_outputs.attentions[-1][0].mean(0)[0, 1:]
    n_patches = attn.shape[0]
    side = int(n_patches ** 0.5)
    attn_map = attn.reshape(side, side).detach().numpy()
    attn_map = np.array(Image.fromarray(attn_map).resize(image.size, Image.BILINEAR))

    fig, axes = plt.subplots(1, 2, figsize=(9, 4.5))
    axes[0].imshow(image); axes[0].axis("off"); axes[0].set_title("Исходное изображение")
    axes[1].imshow(image); axes[1].imshow(attn_map, cmap="jet", alpha=0.5); axes[1].axis("off")
    axes[1].set_title("Карта внимания CLIP (последний слой)")
    plt.show()
